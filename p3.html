P3: カリキュラム共創

AI時代の「両利きの学校」

P2で起きたこと → だからP3が必要

P1で教員がAI共創を体験し、P2でコンセンサス型の全員経営が動き始めた。教員は「自分の意見が学校を動かす」ことを実感している。

そのとき、自然に出てくる問いがある:

「教員の学校運営が変わった。では、生徒の学びをどう変えるか？」

P3は、その問いに応える。 教員自身がコンセンサス型で「自分たちの学校のカリキュラム」を共創するプロセスだ。外部からカリキュラムを納品するのではなく、P2で身につけた集合知の力を使って、教員が自分たちで設計する。

世界観との接続

世界観ページで定義した「自然農モデル」を思い出してほしい。従来の教科学習は「商品作物栽培モデル」——均質な知識を効率的に注入する。探究学習は「自然農モデル」——多様なタネが多様な環境で多様な芽を出す。P3は、この2つを「両利き」で同時に実現する設計だ。教科学習をAIで効率化することで生まれた時間とエネルギーが、探究学習の創造性を支える。

P3で育つコンピテンシー

P3のカリキュラムは、7つのAI PROOFコンピテンシーのすべてを育てる場になる:

  • 教科学習（深化）で → 1. 仕組み化と再投資の基盤を作る
  • 探究学習（探索）で → 3. パターン発見、6. 合意形成と共創、7. 意味の自己生成を育てる
  • 教科×探究の接続で → 2. 越境学習、4. 知の形式知化、5. 仮説駆動が自然に発動する
実証済みの方法

教科学習のAI個別最適化はフィズヨビ（AIエージェントによるテキスト自動作成、AIチューター実装。2026年5月リニューアルオープン予定）で実証済み。探究学習の伴走は、北田朋也（KAEL）による出町桝形商店街×鴨沂高校の地域創生探究プロジェクト、小国高校ふるさとイノベーション関西大会優勝メンター、野辺みなも（みなもラボ）による隠岐ユネスコ世界ジオパークでの探究学習7年間の実践に基づいている。詳細はAbout Usを参照。

1. 設計原理：AI時代の「両利きの経営」を学校に翻訳する

「両利きの経営」（Ambidexterity）とは、既存事業の効率化（深化）と新規事業の創造（探索）を同時に追求する経営戦略。これを学校に翻訳する。

教科学習（深化）探究学習（探索）
目的既知の知識体系の習得未知の問いへの挑戦
AIの役割AIチューターによる個別最適化教師と生徒の集合知で創発
推論の型教科書の体系に沿って学ぶ問いから始まり、必要な知識をJust in Timeで獲得
教師の役割コース設計者・学習の伴走者共に探究するファシリテーター
生徒の学び方AIチューターと対話しながら自分のペースでチームで問いを立て、社会と接続しながら
核心

教科学習はAIが個別最適化する。探究学習はAIを活用し人間の集合知で創発する。
この2つを同時に実現することが、AI時代の「両利きの学校」。
教科学習が効率化されることで生まれた時間とエネルギーが、探究学習の質を高める。

2. 教科学習の設計：AI個別最適化学習プラットフォーム

プラットフォームの構成

「両利き」の深化側を支えるのが、デジタルファシリテーション研究所が提供するAI個別最適化学習プラットフォーム。動画＋テキスト＋AIチューターの組み合わせで、全科目のコース学習を実現する。

要素内容
動画フィズヨビで20年間培った教授法に基づく講義動画
テキスト物理: フィズヨビの既存コンテンツ。物理以外: 教師がワークショップで自作（後述）
AIチューター生徒の理解度に応じて問い返し、深掘りする回折格子型AI
API接続クラウドサービスとして提供。生徒の個人Vaultと接続し、学習ログが蓄積される

田原先生エージェントはたたき台——学校の先生エージェントが育つ

物理科目については、フィズヨビの20年のコンテンツとノウハウがそのまま基盤になる。では物理以外の科目はどうするか。

教師向けワークショップを実施し、教師自身がテキストとAIチュータープロンプトを作成する。

Step やること 何が生まれるか
1. 体験 フィズヨビの物理コースを教師自身が受講し、「動く実物」を体験する 「AIチューターとはこういうものか」という理解
2. 型を学ぶ 田原先生エージェントの設計思想（回折格子型AI＝答えを返すのではなく問い返す）を学ぶ AIチューター設計の「型」の習得
3. 自分で作る Claude Code＋Obsidianで、自分の教科・自分の教え方に合わせたテキストとAIチュータープロンプトを作成 自分の教え方が反映されたテキスト＋AIチューター
4. 先生エージェント誕生 教師の教え方・口癖・得意な説明パターンがプロンプトに抽出される 「○○高校の山田先生エージェント」が生まれる
5. プラットフォームに組み込み 作成したコースをAI個別最適化学習プラットフォームに登録 生徒が「自分の学校の先生のAI」と対話しながら全科目を学べる状態
なぜ「自分たちの先生エージェント」が重要か

外部から持ち込まれたAIチューターは「よそのもの」。教師自身が自分の教え方を反映して作ったAIチューターは「自分たちのもの」。コンピテンシーが「たたき台→現場で検証・更新」であるように、カリキュラムが「たたき台→学校がカスタマイズ」であるように、AIチューターも「田原先生エージェント（たたき台）→その学校の先生エージェント」に育つ。「納品ではなく伴走」がすべてのレベルで一貫している。

教師がコース設計者に進化する

教科学習のAI化は、教師を不要にするのではなく、教師をコース設計者に進化させる。

教師がワークショップで教育的意図を言語化→AIがテキストとチュータープロンプトの叩き台を生成→教師がレビュー・改善→プラットフォームに登録→生徒が使う→フィードバックで改善。このサイクルを事例検討会（P4）で共有し、教師の教え方の暗黙知が学校の集合知になっていく。

3. 探究学習の設計：AIを活用した新規事業開発を学校に翻訳する

企業のAI共創型新規事業開発と同じ構造

探究学習のプロセスは、企業がAIを活用して新規事業を開発するプロセスと構造的に同型だ。

プロセス企業のAI共創型新規事業開発生徒の探究学習
課題の発見現場を歩き、顧客の声を聴き、「まだ解かれていない問題」を見つけるフィールドワークで地域を歩き、当事者の声を聴き、「自分の問い」と出会う
AIで深掘りClaude Codeに現場データを読み込ませ、背景・先行事例・市場を調査取材メモをClaude Codeに読み込ませ、背景・先行事例を調査
仮説と検証仮説を立て、プロトタイプを作り、顧客にぶつけて検証する仮説を立て、再取材・実験で検証する
社会に届ける製品・サービスとして市場に出す。フィードバックで改善ZINE・Web・アプリとして社会に届ける。反響が次の問いに
知恵の蓄積プロセス全体がVaultに蓄積→組織の知恵にプロセス全体がVaultに蓄積→ポートフォリオに
企業AXの実践がモデル

この構造は、ユニオンフォレスト（広島）のDDF新規事業開発で実践中のプロセスそのもの。地域密着型企業が、AIを活用して現場の課題を発見し、仮説を立て、プロトタイプを作り、検証する——このサイクルを高校生が体験する。企業で動いている方法だから、探究学習でも動く。

チェンジメーカー育成プログラムをたたき台とする

P5: チェンジメーカー育成カリキュラムで設計した❶出会う→❷掘る→❸調べる→❹確かめる→❺届ける→❻響くの探究サイクルは、上記の企業新規事業開発プロセスを高校生向けに翻訳したたたき台。各学校がこれを自校の文脈に合わせてカスタマイズする。

重要: チェンジメーカー育成プログラムは「完成品」ではなく「たたき台」。 「出会う」場所はどこか、「届ける」メディアは何か、教科学習との接続点をどう設計するか——これらを教員がP2で身につけたコンセンサス型の力で、自分たちの学校に合わせて共創する。詳細はP5ページを参照。

教科学習と探究学習の接続点：❼「学ぶ」

「両利き」の最大のポイントは、教科学習と探究学習が分離せず接続すること。

探究で「水質調査」をしている生徒
  → 「この水のpHってどうやって計算するんだろう」
  → ❼「学ぶ」が発動
  → AIチューターが化学分析コースをJust in Timeで提示
  → 教科の知識が「今、自分が必要だから」学ぶものに変わる
  → 学んだ知識で❹「確かめる」の精度が上がる

これが「越境学習」（コンピテンシー2）の実装。

JIT学習の実装

探究テーマに応じて、AI個別最適化学習プラットフォームから必要なコースを呼び出す。物理はフィズヨビのコンテンツ、他教科は教師が作った「自分たちの先生エージェント」のコースで学ぶ。

  • 水質調査 → 化学分析コース（化学の○○先生エージェントが伴走）
  • 人口問題 → 統計学入門（数学の△△先生エージェントが伴走）
  • 農業IoT → 電子回路コース（物理の田原先生エージェントが伴走）

4. 学校が自分で作る——P3の核心

P2と同じ原則がここでも貫かれる: カリキュラムを「納品」するのではなく、学校自身がカリキュラムを作り続ける力を育てる伴走をする。

私たちが提供するもの（たたき台）学校が作るもの（自分たちのもの）
田原先生エージェント（AIチューターの「型」）自校の先生エージェント——教師の教え方が反映されたAIチューター
AI個別最適化学習プラットフォーム教師が自作したコースが載った「自分たちの学習プラットフォーム」
チェンジメーカー育成プログラムの「たたき台」（P5）地域特性に合わせてカスタマイズされた自校の探究プログラム
Claude Code＋Obsidianの環境（プラットフォーム）教師のコース設計と生徒の探究が蓄積された学校の知恵の資産
学校がAXすることの意味

AIチューターコースを作れる教師は、AI時代の教育者としてのコア能力を獲得している。チェンジメーカー育成プログラムを自校に合わせてカスタマイズできる学校は、自分たちで新しい学校を作っていける学校になっている。これはツールの導入ではなく、学校という組織のAIトランスフォーメーション。

5. 段階的展開

年教科学習（深化）探究学習（探索）教科×探究の接続
1年目フィズヨビの物理コースを体験→教師向けワークショップで各教科のテキスト＋先生エージェントを作り始めるチェンジメーカー育成プログラムのたたき台で、探究サイクルを1周試行探究で「この知識が必要」→ AIチューターがJust in Timeで提供する体験が始まる
2年目全教科で「自校の先生エージェント」が稼働。教師間の相互レビュー・改善2サイクル目。ZINE制作・社会実装まで到達教科と探究のJIT接続が自然に。「この探究テーマなら、ここで化学を学ぶ」が設計できる
3年目生徒が「自分の学校の先生のAI」と日常的に学ぶポートフォリオが総合型選抜の武器に教科×探究が分離せず「学び」として統合。学校が自走

実施体制: P3の各フェーズは、3つのロール（プロセス設計・ダイアログファシリテーター・AIナレッジファシリテーター）＋事務局で協働して実施。教科学習は教師向けワークショップで先生エージェントの構築を支援、探究学習の伴走は研修チーム全体が担う。

6. まとめ：P3の本質

P3が提供するのは「カリキュラム」ではない。

P3が提供するのは「カリキュラムを自分たちで作り続ける力」。

教科学習
田原先生エージェントという「たたき台」を提供 → 教師がワークショップで自分の教え方を反映 → 「自校の先生エージェント」が生まれる
探究学習
チェンジメーカー育成プログラムという「たたき台」を提供 → 学校が地域特性に合わせてカスタマイズ → 教師と生徒の集合知でプログラムが進化し続ける
教科と探究の接続
♥7「学ぶ」が接続点。探究の中で「今、この知識が必要だ」が生まれる → AIチューターがJust in Timeで教科学習を提供 → 「教えられる知識」が「自分が必要とする知識」に変わる
共通基盤
Claude Code＋Obsidian → 教師のナレッジが蓄積され学校の資産に → 生徒の探究がポートフォリオに → 学校自体がAXされ、自分たちで新しい学校を作り続ける

これがAI時代の「両利きの学校」。
