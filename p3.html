P3：カリキュラム共創プロセス — AI時代の「両利きの学校」

教科学習はAIが個別最適化する。探究学習は人間の集合知で創発する。

P2で起きたこと → だからP3が必要

P1で教員がAI共創を体験し、P2でコンセンサス型の全員経営が動き始めた。教員は「自分の意見が学校を動かす」ことを実感している。

そのとき、自然に出てくる問いがある:

「教員の学校運営が変わった。では、生徒の学びをどう変えるか？」

P3は、その問いに応える。 教員自身がコンセンサス型で「自分たちの学校のカリキュラム」を共創するプロセスだ。外部からカリキュラムを納品するのではなく、P2で身につけた集合知の力を使って、教員が自分たちで設計する。

世界観との接続

世界観ページで定義した「自然農モデル」を思い出してほしい。従来の教科学習は「商品作物栽培モデル」——均質な知識を効率的に注入する。探究学習は「自然農モデル」——多様なタネが多様な環境で多様な芽を出す。P3は、この2つを「両利き」で同時に実現する設計だ。教科学習をAIで効率化することで生まれた時間とエネルギーが、探究学習の創造性を支える。

P3で育つコンピテンシー

P3のカリキュラムは、7つのAI PROOFコンピテンシーのすべてを育てる場になる:

  • 教科学習（深化）で → 1. 仕組み化と再投資の基盤を作る
  • 探究学習（探索）で → 3. パターン発見、6. 合意形成と共創、7. 意味の自己生成を育てる
  • 教科×探究の接続で → 2. 越境学習、4. 知の形式知化、5. 仮説駆動が自然に発動する
実証済みの方法

教科学習のAI個別最適化はフィズヨビ（オンライン物理教育20年、参考書10冊以上）で実証済み。探究学習の伴走は、北田朋也（KAEL）による出町桝形商店街×鴨沂高校の地域創生探究プロジェクト、小国高校ふるさとイノベーション関西大会優勝メンター、野辺みなも（みなもラボ）による隠岐の島でのジオパーク探究学習7年間の実践に基づいている。詳細はAbout Usを参照。

1. 設計原理：AI時代の「両利きの経営」を学校に翻訳する

企業における両利きの経営

「両利きの経営」（Ambidexterity）とは、既存事業の効率化（深化）と新規事業の創造（探索）を同時に追求する経営戦略。AI時代においては：

既存事業（深化）新規事業（探索）
目的今ある価値の最大化まだない価値の創造
AIの役割自動化・効率化（演繹・帰納）集合知の創発（アブダクション）
推論の型昨日のパターンから今日を予測点と点がつながり新しい意味が生まれる
企業AXでの実践業務プロセスのAI自動化現場のナレッジ→集合知→新しい戦略

学校への翻訳

教科学習（深化）探究学習（探索）
目的既知の知識体系の習得未知の問いへの挑戦
AIの役割AIチューターによる個別最適化教師と生徒の集合知で創発
推論の型教科書の体系に沿って学ぶ問いから始まり、必要な知識をJust in Timeで獲得
教師の役割コース設計者・学習の伴走者共に探究するファシリテーター
生徒の学び方AIチューターと対話しながら自分のペースでチームで問いを立て、社会と接続しながら
核心

教科学習はAIが個別最適化する。探究学習は人間の集合知で創発する。
この2つを同時に実現することが、AI時代の「両利きの学校」。
教科学習が効率化されることで生まれた時間とエネルギーが、探究学習の質を高める。

なぜ「両利き」が必要か — 2040年問題との接続

2040年には事務職440万人余剰 vs AI人材340万人不足（経産省推計）。教科学習だけでは「AIに代替される人材」を量産するだけ。探究学習だけでは「基礎のない創造」になる。

両方が揃って初めて、AI PROOFな人材が育つ。

  • 教科学習で「仕組み化と再投資」（コンピテンシー1）の基盤を作る
  • 探究学習で「パターン発見」「合意形成と共創」「意味の自己生成」（コンピテンシー3,6,7）を育てる
  • 両者の接続で「越境学習」「知の形式知化と接続」「仮説駆動と事業開発」（コンピテンシー2,4,5）が自然に発動する

2. 教科学習の設計：AIチューターによる個別最適化

フィズヨビの「型」を提供する

フィズヨビ（田原が20年間運営するオンライン物理教育サービス）で開発されたAIチューターの設計パターンを、教科学習プラットフォームの「型」として提供する。

フィズヨビが提供する「型」教師がやること
AIチューターの設計パターン（体系的に教えるモード＋ソクラテス式で導くモード）自分の教科に応用して、AIチューターのプロンプトを設計
個別最適化の仕組み（受講者カルテ＋躓き点マップ）自分の教科の典型的な躓きパターンを言語化・構造化
教材制作のAIパイプラインClaude Codeを使って自分のコースを作る

Step 1: まずフィズヨビの物理コースをそのまま提供し、AIチューターの「動く実物」を体験。
Step 2: その「型」を各教科の教師が応用し、自分の教科のAIチューターコースを作る。

教師がコース学習を自分で作れるようにする伴走

ここが「両利きの学校」の核心の一つ。教科学習のAI化は、教師を不要にするのではなく、教師をコース設計者に進化させる。

教師がコースを作るプロセス

Step内容AIの支援
1. 教育的意図の言語化「この単元で生徒に何を掴ませたいか」をAI壁打ちで明確化インタビューエージェント（フィズヨビ教材制作Agentic AI設計書の7つの質問軸を活用）
2. カリキュラム構造の設計単元の流れ、前提知識、到達目標の構造化カリキュラム設計エージェントが叩き台を生成
3. 教材テキストの生成教師の意図に基づいた解説テキスト＋演習問題テキスト生成エージェント＋教師のレビュー
4. AIチュータープロンプトの設計「この単元では生徒にこう問いかけてほしい」プロンプト生成エージェント
5. 躓き点マップの作成「ここで生徒はこう間違える」という教師の暗黙知の形式知化教師の経験をAIが構造化
6. 検証と改善実際に生徒に使ってもらい、フィードバックを反映品質検証エージェント

伴走の形

  • 月1回の事例検討会: 「私はこうやってコースを作った」を教師同士で共有
  • Claude Code + Obsidianで蓄積: 各教師のコース設計がMDファイルで蓄積され、学校全体のナレッジに
  • 教師間の相互受粉: 国語教師の「読解の躓き点マップ」が数学教師の「証明の躓き点マップ」のヒントになる
企業AXとの同型性

これはまさにドラッグストアのバイヤーの暗黙知→集合知の構造と同型。

  • ドラッグストア: 個々のバイヤーの棚づくりの暗黙知 → AIで言語化・構造化 → 集合知
  • 学校: 個々の教師の教え方の暗黙知 → AIで言語化・構造化 → 教科横断の集合知

3. 探究学習の設計：教師と生徒の集合知で創発する

チェンジメーカー育成プログラムをたたき台とする

チェンジメーカー育成カリキュラム設計で設計した❶〜❾のサイクルをたたき台として提供し、各学校がこれを自校の文脈に合わせてカスタマイズする。

❶〜❾サイクルの概要

フェーズ内容コンピテンシー
❶ 出会うフィールドワーク、地域探究、ZINE触発意味の自己生成、仮説駆動
❷ 掘る詳細取材、AIに読み込ませて深掘り、MUSVIの窓で遠隔対話知の形式知化、越境学習
❸ 調べる先行事例研究、「既に誰かやってないか」調査パターン発見、仮説駆動
❹ 確かめる再取材・実験、仮説検証仮説駆動、パターン発見
❺ 届けるZINE制作、論文、Web、アプリ開発知の形式知化、仕組み化
❻ 響く社会的反響、キャリア探究意味の自己生成、合意形成
❼ 学ぶJust in Timeのコース学習（教科学習との接続点）越境学習、仕組み化
❽ 探究の質を高めるAI共創×NVC対話でのファシリテーション合意形成、パターン発見
❾ 自分を知るVault起点キャリア探究、ポートフォリオ形成意味の自己生成

「たたき台」としての使い方

重要: チェンジメーカー育成プログラムは「完成品」ではなく「たたき台」。

提供するもの:
  ・❶〜❾のサイクル構造（枠組み）
  ・各フェーズのワークシート・プロンプト集
  ・評価ルーブリック
  ・先行事例（東風高校、長崎県パイロット校）

各学校がカスタマイズするもの:
  ・「出会う」場所はどこか（地域の特性に応じて）
  ・「届ける」メディアは何か（ZINE？Web？地域新聞？）
  ・「響く」相手は誰か（地域住民？企業？自治体？）
  ・教科学習との接続点（❼「学ぶ」）をどう設計するか
  ・何年生でどのフェーズに重点を置くか

教科学習と探究学習の接続点：❼「学ぶ」

「両利き」の最大のポイントは、教科学習と探究学習が分離せず接続すること。

探究で「水質調査」をしている生徒
  → 「この水のpHってどうやって計算するんだろう」
  → ❼「学ぶ」が発動
  → AIチューターが化学分析コースをJust in Timeで提示
  → 教科の知識が「今、自分が必要だから」学ぶものに変わる
  → 学んだ知識で❹「確かめる」の精度が上がる

これが「越境学習」（コンピテンシー2）の実装。

フィズヨビのJIT学習モデル

「田原先生エージェントがフィズヨビのノウハウで、探究テーマに最適化されたコースをAIで生成」

  • 水質調査 → 化学分析コース
  • 人口問題 → 統計学入門
  • 農業IoT → 電子回路コース

4. 学校がプログラムを自分で作る：AXとしての伴走支援

「両利きの学校」は教えてもらうものではなく、自分たちで作るもの

ここがP3の最も重要な設計思想。田原チームがカリキュラムを「納品」するのではなく、学校自身がClaude Code＋Obsidianの環境でカリキュラムを開発できるように伴走する。

これは企業AXと完全に同じ構造:

企業AX学校AX（P3）
外部コンサルがシステムを「納品」→ 使われない外部がカリキュラムを「納品」→ 現場に合わない
現場の声を聞き、現場が自分で作る → 使われる教師と生徒の声から、学校が自分で作る → 実践される
Claude Code＋Obsidianでナレッジ蓄積Claude Code＋Obsidianでカリキュラム蓄積
AIが集合知を構造化AIが教育実践知を構造化

伴走支援の3層構造

層1：教科学習プラットフォームの提供

提供するもの形態学校がやること
フィズヨビのコース（物理）そのまま使える教材生徒に提供。教師は伴走者に
AIチューターの「型」設計パターン＋テンプレート他教科に応用してコース作成
教材制作Agentic AIパイプラインClaude Codeのスキル群自校の教材を自分で作る

層2：教師自身がコースを作ることへの伴走

伴走の内容形態頻度
コース設計ワークショップ教師が自分の教科でAIチューターコースを作る実践学期に1回
事例検討会作ったコースの共有＋改善月1回
躓き点マップの共同作成教科横断で「生徒はどこで躓くか」を構造化継続的
Obsidianへの蓄積教師のコース設計がMDファイルで学校の資産に日常的

層3：学校独自のプログラム開発への伴走

伴走の内容形態進め方
チェンジメーカー育成プログラムの紹介❶〜❾のサイクルをたたき台として共有導入時
地域特性に合わせたカスタマイズ「うちの学校なら❶はどこで出会うか」を教師と対話学期に1回のワークショップ
生徒のフィードバック統合生徒の声をAIで集約→プログラム改善学期末
教師間の集合知形成「やってみてどうだったか」をAIで構造化月1回の事例検討会
他校との相互受粉複数校のナレッジをクロス分析年2回の合同研修

Claude Code＋Obsidian環境の役割

教師のObsidian:
  ├── 教科/
  │   ├── コース設計/（AIチューターコース）
  │   ├── 躓き点マップ/
  │   └── 生徒の学習ログ分析/
  ├── 探究/
  │   ├── 生徒チームの探究テーマ/
  │   ├── フィールドワーク記録/
  │   └── 探究→教科の接続ログ/
  ├── 学校共有/（Git or Sync）
  │   ├── カリキュラム設計書/
  │   ├── 事例検討会のナレッジ/
  │   └── 月次レポート/
  └── CLAUDE.md（学校の教育方針・スクールミッション）

生徒のObsidian:
  ├── 探究ジャーナル/
  ├── AIチューターとの学習ログ/
  ├── ポートフォリオ/
  └── キャリア探究/
学校がAXすることの意味

AIチューターコースを作れる教師は、AI時代の教育者としてのコア能力を獲得している。
チェンジメーカー育成プログラムを自校に合わせてカスタマイズできる学校は、自分たちで新しい学校を作っていける学校になっている。
これはツールの導入ではなく、学校という組織のAIトランスフォーメーション。

5. 段階的展開

年教科学習（深化）探究学習（探索）教科×探究の接続
1年目フィズヨビの「型」で理科→数学→英語と展開。教師がAIチューターコースを作り始めるチェンジメーカー育成プログラムのたたき台で、探究サイクルを1周試行探究で「この知識が必要」→ AIチューターが教科学習をJust in Timeで提供する体験が始まる
2年目全教科でAIチューターコース運用。教師間の相互レビュー・改善2サイクル目。ZINE制作・社会実装まで到達教科と探究のJIT接続が自然に。教師が「この探究テーマなら、ここで化学を教える」と設計できる
3年目生徒がAIチューターを日常的に活用ポートフォリオが総合型選抜の武器に教科×探究が分離せず「学び」として統合。学校が自走

実施体制: P3の各フェーズは、3つのロール（プロセス設計・ダイアログファシリテーター・AIナレッジファシリテーター）＋事務局で協働して実施。教科学習の伴走は田原先生エージェント（AIチューター）、探究学習の伴走は研修チーム全体が担う。

6. まとめ：P3の本質

P3が提供するのは「カリキュラム」ではない。

P3が提供するのは「カリキュラムを自分たちで作り続ける力」。

教科学習
フィズヨビという「型」を提供 → 教師が自分の教科に応用 → AIチューターコースを自分で作れる教師が育つ
探究学習
チェンジメーカー育成プログラムという「たたき台」を提供 → 学校が地域特性に合わせてカスタマイズ → 教師と生徒の集合知でプログラムが進化し続ける
教科と探究の接続
♥7「学ぶ」が接続点。探究の中で「今、この知識が必要だ」が生まれる → AIチューターがJust in Timeで教科学習を提供 → 「教えられる知識」が「自分が必要とする知識」に変わる
共通基盤
Claude Code＋Obsidian → 教師のナレッジが蓄積され学校の資産に → 生徒の探究がポートフォリオに → 学校自体がAXされ、自分たちで新しい学校を作り続ける

これがAI時代の「両利きの学校」。
