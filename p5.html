自分が制作したものが他者に読まれて影響が生じる体験:

  • 「あの記事読んだよ」と地域の人に声をかけられる
  • 他校の生徒が自分のZINEを引用して新しい探究を始める
  • 取材先の当事者から「変化が起きた」と報告が来る
  • 読者から反論や別の視点が届く

ZINEを通して相互に影響を与え合いながら学んでいく環境が生まれる:

  • 後輩がZINEを読んで新しいテーマと出会う（→ ❶へ）
  • 他校のZINEと自分のZINEが干渉して新しい問いが生まれる
  • 地域の課題がZINEを通じて可視化され、行政や企業が動き始める

梅田の関与（❻）: 社会的反響を受けた後が梅田の最も重要な出番。Vaultに蓄積された探究の軌跡と社会からのフィードバックを手がかりに、「この経験を通じて、自分は何をしたい人間だと分かったか」「次にどこへ進みたいか」を掘り下げる。これがキャリア探究であり、総合型選抜の志望理由書の核になる。

4. 技術環境——個人Vault＋Claude Code

4.1 環境の構成

PowerShellを開いて自分の名前を入力すると、自分のVaultでClaude Codeが起動する。

> .\start.ps1 田中花子
→ C:\Students\田中花子\Vault\ に移動
→ Claude Codeが起動
→ CLAUDE.mdが探究コンテキストを読み込む

4.2 個人Vaultの構造

C:\Students\田中花子\Vault\
├── CLAUDE.md           ← 探究の文脈・方針
├── 00-探究ノート/       ← 日々の気づき・取材メモ
├── 01-先行事例/         ← 調べた事例の整理
├── 02-仮説と検証/       ← 仮説→検証→修正の記録
├── 03-アウトプット/     ← ZINE原稿、Webサイト、アプリ、論文
├── 04-JIT学習/          ← コース学習の記録
├── 回折ジャーナル/      ← 想定と現実のズレ
└── .git/               ← 版管理（3年間の思考の軌跡）

4.3 この環境で行うこと

フェーズVaultでやること
❷掘る取材メモを入力→AIが背景を掘り下げ→ノートとして蓄積
❸調べる「先行事例は？」とAIに聞く→事例をノートに整理→[[内部リンク]]で接続
❹確かめる仮説と検証結果を記録。プロトタイプもClaude Codeで作る
❺届けるZINE / Webサイト / アプリ / 論文をこの環境で制作
❼学ぶ田原先生エージェントがコース学習を生成→学習ノートとして蓄積

4.4 なぜ個人Vaultなのか

特性教育的な意味
すべてが1箇所に集まる探究の全プロセスが蓄積。断片化しない
[[内部リンク]]で接続取材ノートと先行事例と仮説が有機的につながる
Gitで履歴が残る3年間の思考の軌跡がポートフォリオになる
CLAUDE.mdでコンテキスト共有AIが「この生徒の探究テーマ」を理解した上で支援
アウトプットも同じ環境「考えたこと」と「作ったもの」が分離しない
梅田メソッドの素材になるVaultを開けば3年間の関心の変遷が見える
卒業時に持ち出せるフォルダごとコピー。ベンダーロックインなし

4.5 教員の環境

教員も同じ構造の個人Vaultを持つ。教員も生徒も同じOS——馬場さんの「3つのレベルで同じ構造変化が必要」の技術的実装。
