Claude Code＋Obsidian 環境解説

思考を積み上げていくAI活用プラットフォーム
本ノートの目的

NEXTハイスクール構想で導入するAI活用プラットフォーム「Claude Code＋Obsidian」について、教育委員会・学校管理職・教員に向けて解説する。「なぜこの組み合わせなのか」「何ができるのか」「どう段階的に導入するのか」を示す。

プロセス全体の中での位置づけは プロセスデザイン全体設計 を参照。

1. なぜ「思考を積み上げる」環境が必要か

1.1 従来のAI活用の問題

【従来のAI活用（ChatGPT等）】 質問 → AIが回答 → 読んで終わり → 消える 質問 → AIが回答 → 読んで終わり → 消える 質問 → AIが回答 → 読んで終わり → 消える → 毎回ゼロから。対話の蓄積がない。 → 「先週考えたこと」をAIは知らない。 → 思考が使い捨てになる。

1.2 Claude Code＋Obsidianの世界

【Claude Code＋Obsidian】 思考 → Vaultに記録 → AIが文脈を理解 → 深掘り → Vaultに追記 ↑ │ └──────── 次の対話で「前回の続き」から始まる ──────────┘ → 思考が積み上がる。 → AIは「この人が何を考えてきたか」を知っている。 → 1ヶ月前のノートと今日のノートがつながる。

核心: AIとの対話が「使い捨て」から「蓄積」に変わる。 これが教員の授業設計にも、生徒の探究学習にも、学校の組織運営にも共通する価値。

2. 3つの構成要素

2.1 Obsidian — 思考の倉庫

┌─────────────────────────────────────┐ │ Obsidian Vault（自分の思考の倉庫） │ │ │ │ 授業設計メモ.md │ │ 生徒Aの探究テーマ.md │ │ 研修での気づき.md │ │ 先行事例調査.md │ │ │ │ すべてがマークダウンファイル │ │ [[内部リンク]]でつながる │ └─────────────────────────────────────┘
特徴意味
マークダウンファイル特殊な形式ではなく、テキストファイル。どの環境でも開ける
ローカル保存クラウドに依存しない。自分のPCにある
[[内部リンク]]ノート同士をつなげられる。「授業設計メモ」と「生徒の探究テーマ」がリンクでつながる
検索が速い数千件のノートでも瞬時に検索できる
無料ソフトウェア利用料がかからない

教員にとっての価値: 授業のアイデア、研修の気づき、生徒の様子 — これまでバラバラだった教員の思考が、1つの場所に蓄積され、つながる。

2.2 Claude Code — 思考のパートナー

┌─────────────────────────────────────┐ │ Claude Code │ │ │ │ ターミナル（黒い画面）で動くAI │ │ │ │ 教員: 「明日の化学の授業を │ │ 探究テーマと接続したい」 │ │ │ │ Claude: Vaultの中の │ │ ・生徒の探究テーマ一覧 │ │ ・教科書の進度 │ │ ・前回の授業の振り返り │ │ を読んだ上で提案する │ └─────────────────────────────────────┘
特徴意味
Vaultを読めるAIが「この人の文脈」を理解した上で応答する
Vaultに書けるAIとの対話結果をノートとして保存できる
CLAUDE.mdVault直下に置く設定ファイル。「この学校の方針」「この教科の特徴」などを書いておくと、AIがそれを踏まえて応答する
ファイル操作ができるテキスト生成だけでなく、ファイルの作成・編集・検索ができる

教員にとっての価値: 「自分の授業設計の文脈を分かっているAI」と壁打ちできる。毎回ゼロから説明しなくていい。

2.3 Git — 思考の履歴

┌─────────────────────────────────────┐ │ Git（版管理） │ │ │ │ 3月1日 → 授業設計v1 │ │ 3月5日 → 授業後の振り返りを追記 │ │ 3月10日 → 生徒のフィードバックで修正 │ │ 3月15日 → v2として大幅改訂 │ │ │ │ → いつ、何を、なぜ変えたかが │ │ すべて記録される │ └─────────────────────────────────────┘
特徴意味
変更履歴が残る「いつ、何を、なぜ変えたか」が全て記録される
いつでも戻れる「3月5日の状態に戻す」ができる。失敗を恐れずに変更できる
思考の軌跡が見える生徒の3年間の関心の変遷、教員の授業設計の進化が時系列で見える

教員にとっての価値: 授業改善の軌跡が自動的に記録される。

生徒にとっての価値: 3年間の探究の軌跡がそのままポートフォリオになる。

3. 具体的な使用場面

3.1 教員が使う場面

場面A: 授業設計

教員: 明日の物理の授業、力学的エネルギーの導入なんだけど、
      生徒の探究テーマに接続する方法を考えたい。

Claude: Vaultの生徒の探究テーマ一覧を確認しました。
        ・田中さん: 水質調査 → 位置エネルギーと水力の接続が可能
        ・佐藤さん: 離島の交通 → 運動エネルギーと燃費の接続
        ・山田さん: 農業IoT → エネルギー変換効率の接続

        前回の授業振り返りノートでは「数式だけだと理解が浅い」と
        書かれていたので、実例から入る構成を提案します…

→ AIが「この教員のVault」を読んで、文脈を踏まえた提案をする
→ 授業案がVaultに保存される → 次回の授業設計の参考になる

場面B: 研修の振り返り

教員: 今日の事例研究会で出た意見を整理して。

Claude: Googleフォームに集まった12名の意見を分析しました。
        4つのクラスターに分類:
        1. AI壁打ちの具体的な使い方への関心
        2. 評価方法への懸念
        3. 教科横断の可能性への期待
        4. 時間確保の現実的な課題

        前回の事例研究会の記録と比較すると、
        「評価方法への懸念」が新たに浮上しています。

→ 過去の記録と接続した分析ができる
→ KJ法の結果がVaultに蓄積 → 学校の知識基盤になる

場面C: 職員会議の記録

教員: 今日の職員会議のコンセンサス評価を記録して。

Claude: 議題「来年度の探究学習の時間配置」
        第1回コンセンサス評価:
        ・賛成(1): 8名
        ・懸念あり(2): 4名 — 理由: 教科の時間確保との両立
        ・ブロック(3): 1名 — 理由: 評価基準が未定

        前回の議題「AI壁打ちの導入範囲」の評価経過と
        リンクしておきます。

→ 意思決定の経緯が記録される → 「なぜこう決めたか」が後から追える

3.2 生徒が使う場面

場面D: 探究の深掘り

生徒: 今日のフィールドワークで、漁師のおじいちゃんに話を聞いた。
      メモを整理したい。

Claude: 取材メモを構造化しました。
        ■ 漁獲量の変化（数字データ）
        ■ 海水温の体感的な変化
        ■ おじいちゃんの「昔はこうだった」エピソード

        先行事例フォルダにある[[海洋環境変化の調査レポート]]と
        照合すると、おじいちゃんの体感と科学データが一致しています。

        仮説: 「海水温の上昇が漁獲パターンを変えている」
        次のステップ: 過去20年の海水温データを入手して検証？

→ 1次情報（取材メモ）と2次情報（先行事例）がVault内でつながる
→ 仮説が生まれ、次のアクションが見える

場面E: ZINE原稿の作成

生徒: 探究の成果をZINEにまとめたい。

Claude: Vaultに蓄積された探究ノートから、以下の構成を提案:
        1. 「おじいちゃんの海」— 取材エピソードから始める
        2. 「データが語ること」— 海水温×漁獲量の分析
        3. 「2040年の海」— 仮説と検証結果
        4. 「私たちにできること」— 提言

        [[取材メモ_漁師インタビュー]]と[[海水温データ分析]]から
        使える素材をリストアップしました。

→ Vaultに蓄積した素材から、アウトプットを組み立てる
→ Claude Codeでレイアウト・校正まで支援

場面F: 自分を知る（キャリア探究）

梅田: Claude、この生徒のVaultの関心の変遷を時系列で要約して。

Claude: 田中花子さんのVault（Git履歴含む）を分析:
        1年4月: 海が好き → 漁業に興味
        1年9月: フィールドワークで「海が変わった」に気づく
        1年12月: 海洋環境問題に焦点を移す
        2年4月: データ分析に没頭（JIT学習で統計学入門を受講）
        2年9月: 政策提言に興味 → 行政とのつながりを求め始める

        繰り返し出てくるテーマ: 「現場の声」と「データ」をつなげること

梅田: 田中さん、「現場の声とデータをつなげる」って、
      なぜそこに惹かれるんだと思う？

→ 3年間の思考の軌跡が見えるから、深いキャリア探究ができる

4. CLAUDE.md — AIに文脈を伝える設定ファイル

Vault直下に置くCLAUDE.mdは、Claude Codeが「この環境で何を知っておくべきか」を理解するための設定ファイル。

4.1 教員のCLAUDE.md（例）

# 東風高校 物理科 山田先生のVault

## 担当
- 物理基礎（1年）、物理（2年選択）
- 探究学習アドバイザー（2年3組担当）

## 授業方針
- 「なぜ？」から始める授業設計
- AI壁打ちを生徒に使わせる（パターンA）
- 探究テーマとの接続を意識する

## 今年度の探究テーマ（担当生徒）
- 田中花子: 海洋環境と漁業
- 佐藤太郎: 離島の再生可能エネルギー
- 山田次郎: 農業IoTとセンサー技術

## 参照すべきファイル
- 授業設計/ — 各授業の設計メモ
- 探究支援/ — 生徒の探究に関する記録
- 研修記録/ — 事例研究会の記録

4.2 生徒のCLAUDE.md（例）

# 田中花子の探究Vault

## 探究テーマ
海洋環境変化が地域漁業に与える影響

## 現在のフェーズ
調べる（先行事例研究→仮説構築）

## これまでの経緯
- 1年4月: 地元の漁港でフィールドワーク
- 1年9月: 漁師へのインタビュー実施
- 1年12月: 海水温データの収集開始

## 仮説
海水温の上昇が回遊パターンを変え、
小規模漁業の漁獲構成を変化させている

## 次のステップ
過去20年の海水温データと漁獲統計を照合して仮説を検証

## 学習の進度
- 統計学入門（JIT学習）: 第5章まで完了
- 海洋科学の基礎: 進行中

5. 技術的な構成

5.1 必要なソフトウェア

ソフトウェア用途費用
Obsidianノート管理・閲覧・編集無料（教育利用）
Claude CodeAI壁打ち・ファイル操作有料（Anthropic契約）
Git版管理無料
VS Code（任意）テキスト編集の補助無料

5.2 動作環境

項目要件
OSWindows 10/11, macOS, Linux
ストレージテキストファイルのみなので、数GB程度で十分
ネットワークClaude Code使用時にインターネット接続が必要
PC学校の既存PCで動作可能。高スペック不要

5.3 起動方法

教員の場合:

1. ターミナル（PowerShell）を開く
2. 自分のVaultフォルダに移動
3. claude と入力してEnter
→ Claude Codeが起動。Vault内のファイルを読み、CLAUDE.mdを理解した状態で対話開始

生徒の場合:

1. ターミナルを開く
2. .\start.ps1 田中花子 と入力
→ 自分のVaultフォルダに移動
→ Claude Codeが起動
→ CLAUDE.mdが探究コンテキストを読み込んだ状態で対話開始

6. セキュリティとプライバシー

懸念対応
生徒のデータがクラウドに流出しないかVaultはローカルPC上のファイル。クラウド同期は任意設定。Claude Codeとの対話はAnthropicのAPI経由だが、学習データには使用されない
生徒同士が互いのVaultを見られないか個人Vaultは個人のフォルダに保存。OSのユーザー権限で分離
教員が生徒のVaultを見る必要があるかキャリア面談時に生徒の許可を得てVaultを参照。普段は分離
個人情報の取り扱い取材メモに個人情報が含まれる場合は匿名化のルールを設定。CLAUDE.mdに記載
AIの回答の正確性Claude Codeは間違えることがある。生徒にはAIの回答を鵜呑みにせず検証する姿勢を教える（これ自体が教育的）

8. 他のAIツールとの比較

項目ChatGPT / Gemini等Claude Code＋Obsidian
対話の蓄積チャット履歴は残るが構造化されないVaultに構造化された形で蓄積。[[内部リンク]]で接続
文脈の理解毎回の対話は独立（長い履歴は忘れる）CLAUDE.mdとVault内のファイルで「この人の文脈」を常に理解
ファイル操作限定的（添付ファイルの読み込み程度）Vault内の全ファイルを読み書き。新規ファイル作成、検索、編集が可能
版管理なしGitで全変更履歴を記録
複数ノート間の接続不可能[[内部リンク]]で自由に接続
カスタマイズプロンプトで都度指示CLAUDE.mdに方針を書けば、毎回の対話で自動的に反映
コスト月額課金（個人単位）API利用料（使用量に応じた従量課金。学校単位で契約可能）
データの所在クラウド上（提供元のサーバー）ローカルPC上（ファイルは自分のPC）
卒業時の持ち出しエクスポートが困難フォルダごとコピー。ベンダーロックインなし

9. よくある質問

Q1: 教員のITスキルが低くても使えるか？

使える。最初のセットアップは北田が支援する。日常の使用は「ターミナルでclaudeと打つ → 日本語で話しかける」だけ。マークダウンの書き方は最低限で十分（太字とリンクだけ覚えればいい）。Stage 1でコアチームが「翻訳者」の役割を果たし、Stage 2では教員同士で教え合う。

Q2: 学校のPCで動くのか？

動く。Obsidianはテキストエディタなので軽い。Claude Codeもターミナルアプリケーションなので高スペック不要。インターネット接続があればOK。

Q3: ChatGPTではダメなのか？

ChatGPTでは「思考の積み上げ」ができない。毎回の対話が独立し、ノート間のリンクも、版管理も、CLAUDE.mdによる文脈共有もない。「1回だけの質問」ならChatGPTでも十分だが、「3年間かけて探究を積み上げる」にはClaude Code＋Obsidianの構造が必要。

Q4: 費用はどのくらいかかるか？

  • Obsidian: 無料
  • Git: 無料
  • Claude Code: Anthropicの利用料（詳細は契約形態による）

学校単位での契約については、導入計画の中で具体的に見積もる。

Q5: 生徒のプライバシーは守れるか？

Vaultはローカル保存。生徒のVaultは個人フォルダに分離。Claude CodeのAPI通信は暗号化されており、Anthropicはデータを学習に使用しない。取材メモ中の個人情報は匿名化ルールを設定する。
