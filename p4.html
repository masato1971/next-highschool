P4: 実践伴走

事例研究会で「両利きの学校」を育てる

P3で設計した → P4で教室を変える

P1で教員がAI共創を体験し、P2で学校運営がコンセンサス型に変わり、P3で「両利きの学校」のカリキュラムを設計した。しかし、設計だけでは教室は変わらない。

P4は、「両利きの学校」の多様な実践——AIチューターによる教科学習、AI共創型の授業、探究学習の伴走——を事例研究会で共有し、学校全体の集合知として育てる伴走プロセスだ。

P4で育つコンピテンシー

P4の多様な実践は、AI PROOFコンピテンシーを生徒が日常的に体験する場になる:

  • AIチューター活用 → 1. 仕組み化と再投資: AIに基礎学習を任せ、自分は問いに集中する
  • AI統合型授業 → 6. 合意形成と共創 + 4. 知の形式知化: 多様な声を統合する
  • 探究×教科の接続 → 2. 越境学習 + 5. 仮説駆動: 教科の枠を越えて学ぶ

2. P4が伴走する実践の領域

P3の「両利きの学校」に対応して、P4は教科学習（深化）と探究学習（探索）の両方の実践を伴走する。

3. 事例研究会を核とした伴走

事例研究会が学校を変える

P4の核は月1回の事例研究会。教員が自分の実践を持ち寄り、共有し、AIで構造化して学校全体の知恵にする。「やり方を教えれば自分でできる」は幻想。やってみた → 共有した → フィードバックをもらった → もっと良くなった、このサイクルが文化になることで、学校全体が変わる。

事例研究会の構造（月1回・1時間）

実践報告（2名 × 10分）

「AIチューターでこの単元をやってみた」「探究学習でAIを使って取材を深掘りした」「教科と探究を接続する授業を試した」——教科学習でも探究学習でも、どんな実践でも持ち込める。

対話（20分）

「自分の教科ならこう応用できる」「探究でも同じ構造が使える」——教科学習と探究学習の実践者が混ざって対話することで、「両利き」の接続が自然に生まれる。

AI統合＋パターン抽出（20分）

対話の内容をClaude Codeで構造化。うまくいった実践の成功パターンを命名し、保管庫（Vault）に蓄積。回を重ねるごとに「学校の教育実践ライブラリ」が育つ。

事例研究会が「両利き」を接続する

教科学習のAIチューター実践と探究学習のAI活用実践が、同じ事例研究会で共有される。数学教員のAIチューター活用が探究学習の伴走者にヒントを与え、探究のフィールドワーク手法が教科の授業設計に還流する。事例研究会が、P3で設計した「両利き」を現場で結びつけるハブになる。

伴走の段階的移行

事例研究会を核としながら、外部の伴走を段階的に教員自身に移行する。

段階事例研究会の主体外部チームの関与
立ち上げ期研修チームがファシリテーション＋AI構造化モデル実践を教員と共同で実施し、最初の事例を作る
展開期教員が実践を持ち寄り始める。研修チームは対話のファシリテーション必要に応じて教室に入り、個別の実践を伴走
自走期教員が自分たちで運営。AIナレッジファシリテーターのスキルも教員に移転年2回のアドバイザリー＋必要時の相談

4. P1〜P4は同じ原理で動く

P4の4段階（モデル授業→ペア伴走→相互伴走→自走）は、P1（体験→実践→共有→自走）、P2（コアチーム→声の収集→コンセンサス→集合知）、企業AXと同じ原理で動く。すべてに共通するのは「外部が見せる → 一緒にやる → 教員が主導する → 完全自走」の段階的移行。

P4の位置づけ

P4は「教員の日常の授業」という最も地続きな部分を変えるプロセス。ここが変わらなければ、研修（P1）も組織（P2）もカリキュラム（P3）も絵に描いた餅になる。

実施体制: P4のモデル授業・事例検討会は、3つのロール（プロセス設計・ダイアログファシリテーター・AIナレッジファシリテーター）で協働して実施。

7. P4の本質

P4が実現するのは「特別な授業」ではない。

P4が実現するのは「学校全体が学び合うコミュニティになる」こと。

AIチューターの実践も、AI統合型の対話授業も、探究学習の伴走も——すべてが事例研究会に持ち込まれ、教員同士の対話とAI構造化を通じて学校の集合知になる。

「両利きの学校」は、事例研究会という場で育つ。
