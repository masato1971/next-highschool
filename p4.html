P4：授業実践伴走 — 日常の授業をAI共創型に転換する

研修で体験しただけでは、自分の教室は変わらない。一緒にやる。見せ合う。そして自分の授業になる。

1. P4が解く問い

「50人の先生がいても取り組むのは2-3人。30時間の授業のうち3-4時間しか変わらない。20時間、できれば30時間全部変わってほしい。」
——馬場さん（長崎県教育委員会 高校教育課）

P1研修で教員は「体験」する。しかし体験だけでは「自分の授業」は変わらない。P4は、体験を日常の授業実践に転換し、2-3人ではなく50人、3-4時間ではなく30時間を変えるプロセス。

2. AI共創型授業の3パターン

P4で教員が実践するAI共創型授業には、3つの基本パターンがある。教科・単元・生徒の状況に応じて使い分ける。

パターンA：AI壁打ち型

項目内容
教員の役割ゴール設定者＋観察者。「何を考えさせたいか」を設計し、プロセスを見守る
生徒の学び自分のペースでAIと対話しながら理解を深める。躓いたらAIチューターが支援
適する場面概念理解、問題演習、個人の思考の深掘り
P1との接続P1第1回「AIアプリを作る」で教員が体験済みの構造
長崎研修で紹介された大分の数学教師の事例

「正解を言わない先生。やり方は何でもいい。教科書調べてもいい、友達に聞いてもいい、AI使ってもいい、座席自由移動。」

→ 前方は先生と従来型、中央はAIを家庭教師に、後方は4人グループで対話。
「学級崩壊ではなく、教室にすがすがしい学びの空気が流れていた。」

パターンB：意見統合型

項目内容
教員の役割ファシリテーター。問いを投げかけ、統合された全体像をもとに対話を導く
生徒の学び「自分の意見が全体に反映される」体験。多様な視点に触れる
適する場面論述・小論文、倫理的問い、歴史的解釈、理科の仮説検討
P1との接続P1第2回「多様な意見をAIで統合する」で教員が体験済みの構造

パターンC：探究接続型

項目内容
教員の役割接続のコーディネーター。「この化学の知識が、あなたの水質調査にどうつながるか」を架橋する
生徒の学び教科学習が「教えられるもの」から「自分が必要とするもの」に変わる
適する場面探究テーマが明確になった後の教科授業。P5と並行する時期
P3との接続P3カリキュラム共創のJust in Time学習の実装

3パターンの使い分け

状況推奨パターン
新しい概念の導入、演習A（AI壁打ち型）
正解のない問い、議論が必要な内容B（意見統合型）
探究学習が進んでいる時期の教科授業C（探究接続型）
最初の一歩（教員が不慣れ）A（最もシンプル）

3. 伴走の4段階：モデル授業→ペア伴走→相互伴走→自走

なぜ4段階が必要か

「やり方を教えれば自分でできる」は幻想。P1研修で体験しても、自分の教室で一人でやるとなると不安が先に立つ。一緒にやる → 見せ合う → 自分でやる という段階を踏むことで、50人の教員が「自分の授業」として取り組めるようになる。

Stage 1：モデル授業

項目内容
やること研修チームが教員と一緒に教室に入り、AI共創型授業を実施する
ポイント教員は「見学」するのではなく「一緒にやる」。教室の空気の中で体を通して理解する
対象コアチーム教員（5〜10名）× 各自の教科
所要1教員あたり2〜3回のモデル授業（1ヶ月程度）
Vault活用授業設計と振り返りをObsidianに記録。「何がうまくいって、何が想定と違ったか」

モデル授業の流れ（1回分）

事前（30分）
  • 教員と研修チームで授業設計
  • 「この単元で、どのパターンを使うか」を決める
  • AIチューターのプロンプト/フォームの準備
授業（50分）
  • 教員と研修チームが共同で実施
  • 研修チームがファシリテーションの「型」を見せる
  • 教員は生徒の反応を観察しながら一緒に動く
振り返り（30分）
  • 「生徒はどこで動き出したか」「どこで止まったか」
  • AIで振り返りを構造化し、Vaultに記録
  • 次回の改善点を決める

Stage 2：ペア伴走

項目内容
やることモデル授業を経験した教員（メンター）が、未経験の教員とペアを組み、授業設計→実施→振り返りを一緒にやる
ポイントメンターは「教える」のではなく「一緒に考える」。外部の専門家ではなく、同じ学校の同僚が伴走者になる
対象コアチーム教員（メンター）× 次のグループ教員（5〜10名）
所要1ペアあたり2〜3回のペア授業（1〜2ヶ月）
Vault活用ペアでVaultを見せ合いながら授業設計。メンターの授業記録が「こうやればいい」の参考に
企業AXとの同型性
企業（ユニオンフォレスト）学校（P4）
AI研修を受けた部長 → 部下に壁打ちの仕方を見せるモデル授業を経験した教員 → 同僚に授業の仕方を見せる
上司が「教える」のではなく「一緒にやる」メンターが「教える」のではなく「一緒に考える」
成功パターンがVaultに蓄積授業設計がVaultに蓄積

Stage 3：相互伴走

項目内容
やること教員同士が互いの授業を見合い、事例検討会でフィードバックし合う
ポイント「教える-教えられる」の関係を超えて、対等な実践者同士の学び合いに
対象全教員
所要月1回の事例検討会 ＋ 学期に2〜3回の相互授業観察
Vault活用AIが授業記録を構造化し、教科間の接続を可視化。「数学のこの工夫は、理科でも使えるのでは」

事例検討会の構造（月1回・1時間）

実践報告（2名 x 10分）
  • 「この単元でパターンBをやってみた」
  • 「生徒はこう反応した」「想定外だったこと」
対話（20分）
  • 「自分の教科ならこう応用できる」
  • 「あの場面、自分ならこうする」
AI統合（10分）
  • 対話の内容をClaude Codeで構造化
  • 「今月の学び」を可視化
パターン抽出（10分）
  • うまくいった授業の成功パターンを命名
  • Vaultに蓄積 → 学校の共有知に
職人の暗黙知を集合知に — ドラッグストアのバイヤーとの同型性
  • ドラッグストア: うまくいった棚の成功事例を共有 → AI構造化 → 集合知
  • 学校P4: うまくいった授業の成功事例を共有 → AI構造化 → 集合知

どちらも「職人の暗黙知を言語化して共有する」同じ原理。

Stage 4：自走

項目内容
やること教員が自分のペースでAI共創型授業を日常化。外部伴走は後方支援に
ポイント自走は「一人でやる」ではなく「学校のコミュニティの中で進化し続ける」こと
指標30時間中、何時間がAI共創型になっているか
Vault活用Vaultに蓄積された授業設計が「学校の教育実践ライブラリ」に。新任教員の立ち上がりが早くなる

4. 3つの同型構造 — 教室・学校・企業で同じOSが動く

P4の設計原理は、P1（教員研修）・P2（学校組織開発）・企業AXと同じ構造。

段階 P1（教員研修） P4（授業実践） P2（学校組織） 企業AX
SEED 2040年問題の認識 モデル授業（「こうやるのか」の体験） 校長のビジョン提示 経営課題の認識
ROOT 個人のAI壁打ち体験 ペア伴走（「自分でもやってみる」） 全教員のAI壁打ち（声の集約） 個人のAI壁打ち
BRANCH 研修内での対話・統合 相互伴走（「見せ合って学び合う」） コンセンサス型の意思決定 部署内の事例研究会
FOREST 研修の気づきを学校に持ち帰る 自走（「学校全体の日常になる」） 組織文化として定着 全社への横展開
馬場課長の「一気通貫で同じOS」

生徒・教員・教育委員会の3層で同じ構造変化を起こす。
P4はそのうち「教員の日常の授業」という最も地続きな部分を変える。
ここが変わらなければ、研修（P1）も組織（P2）もカリキュラム（P3）も絵に描いた餅になる。

7. P4の本質

P4が実現するのは「特別な授業」ではない。

P4が実現するのは「日常の全ての授業が変わる」こと。

研修で体験した（P1）→ カリキュラムを設計した（P3）→ でも自分の教室で一人でやるのは不安だ

P4はその不安を4段階で解消する:

モデル授業 「こうやるのか」を体で知る
ペア伴走 「隣にいてくれるなら自分もできる」
相互伴走 「見せ合えばもっと良くなる」
自走 「これが自分の授業だ」

この段階を経て、馬場課長の問い「30時間中30時間を変える」が現実になる。
