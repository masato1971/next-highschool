P1: 教員向け研修プログラム

WS＋3回研修＋事例検討会で、授業と職員会議のAI活用土台を作る
概要

NEXTハイスクール構想の最初のフェーズ。教員がAI共創型の授業と学校経営を実践するための土台を作る。2040年問題の共有ワークショップ（1回）＋AI反転授業研修（3回）＋事例検討会（継続）の構成。

成果: WSと3回の研修を通じて、以下の2つの土台が完成する。

  1. 授業のAI活用 — 教員が自分の授業にAI壁打ち・AI統合を組み込める
  2. 職員会議のAI活用 — コンセンサス型意思決定を職員会議に導入できる

WS: 2040年問題の共有ワークショップ（2時間）

このWSの目的

「なぜAIを使うのか」の共通認識を教員全体で作る。2040年の社会像を自分ごととして考え、そこから逆算して「自分の授業をどう変えるか」の問いを持って帰る。

事前準備

対象やることツール
参加教員NEXTハイスクール構想の概要動画（15分程度）を視聴動画配信
参加教員AI壁打ちアプリで「2040年に高校教育はどう変わるべきか」について自分の考えを深掘りAI壁打ちアプリ（Claude等）
参加教員壁打ちで深掘りした考えをテキストで提出Googleフォーム
田原・北田提出されたテキストをClaude Codeで事前に構造化・分析Claude Code

当日のフロー

時間内容方法ファシリテーター
0:00–0:10チェックイン1人30秒で「今の気持ち」を共有田原
0:10–0:30講義: 2040年の社会像ホワイトカラー中間層の縮小、アドバンスド・エッセンシャルワーカー、文理融合型人材の需要。統合整理のセクション1をベースに田原
0:30–0:45グループ対話（3–4人）「2040年の社会で、自分が教えている教科は何のために存在するか？」北田（NVC対話設計）
0:45–0:55意見入力スマホ/PCからGoogleフォームに入力:「2040年に向けて、自分の授業で最も変えるべきことは何か？」—
0:55–1:15AI統合・可視化（ライブ）Claude Codeで全意見をリアルタイムKJ法統合。スクリーンに投影しながらクラスターが浮かび上がる過程を全員で見る北田（AIナレッジファシリテーター）
1:15–1:35コンセンサス評価統合結果から生成された「学校変革の3つの方向性」叩き台に対し、1（賛成）/ 2（懸念あり）/ 3（ブロック）で評価。AIエージェント（生徒・保護者・慎重派教員）の声も追加 → バージョンアップ田原
1:35–1:50全体共有バージョンアップされた方向性を全員で確認。「自分の発言が反映されている」体験田原
1:50–2:00チェックアウト1人30秒で「持ち帰ること」を共有田原

このWSで教員が体験すること

体験意味
AI壁打ちで自分の考えを深掘りする事前課題で「AIと対話する」体験。これが反転授業の事前学習と同じ構造
全員の意見がAIで統合される「声の大きい人の意見が通る」のではなく、全員の声が構造化される体験
コンセンサス評価でバージョンアップする批判が「提案を良くする力」になる体験。職員会議への導入イメージ
AIエージェントが不在の声を代弁する生徒・保護者・慎重派の視点が自動的に入る仕組み
WSの裏の目的

このWS自体が、第1回〜第3回の研修で学ぶことの実体験版。教員は「教えられる」のではなく「体験する」ことで、研修への動機が生まれる。長崎県教育委員会の馬場さんが「本当の意味での民主主義が実現していく夢を抱かせる話だった」と評した体験を、学校の教員全体に広げる。

第1回: AIアプリを作る（オンライン2時間）

この回の目的

教員が自分の授業で使える「AI壁打ちアプリ」を実際に作る。「AIを使える」から「AIアプリを作れる」への跳躍。

事前準備

対象やること
参加教員WSで体験したAI壁打ちを、自分の授業テーマで1回やってみる

当日のフロー

時間内容方法
0:00–0:10チェックイン＋事前課題の振り返りラウンド
0:10–0:30講義: AI壁打ちアプリの設計思想「答えを返すAI」（鏡）vs「問い返すAI」（回折格子）。回折格子型AIが生徒の思考に差異のパターンを生む
0:30–1:10ハンズオン: 自分の授業用AI壁打ちアプリを作るClaude Codeまたはノーコードツールで、自分の教科に最適化した壁打ちプロンプトを設計。例: 「数学の証明を考えさせるAI」「英語エッセイの構成を問い返すAI」「理科実験の仮説を掘り下げるAI」
1:10–1:30ペアで試用作ったアプリを隣の教員に使ってもらい、フィードバック
1:30–1:50改善＋共有フィードバックを反映して改善。3–4名が全体に共有
1:50–2:00チェックアウト＋次回までの課題次回までに自分の授業で1回使ってみる

教員が持ち帰るもの

  • 自分の教科用のAI壁打ちアプリ（プロンプト）
  • 生徒に使わせるための運用イメージ
  • 次回までに授業で1回使うという具体的なアクション

第2回: 多様な意見をAIで統合する（オンライン2時間）

この回の目的

授業やHRで、生徒の意見をAIで統合し、全員が参画する合意形成を実施できるようになる。これが職員会議への導入の土台にもなる。

事前準備

対象やること
参加教員第1回で作ったAI壁打ちアプリを授業で使った経験を振り返り、メモにまとめる

当日のフロー

時間内容方法
0:00–0:15実践報告第1回後に授業でAIを使った体験を3–4名が共有
0:15–0:35講義: AI統合ファシリテーションGoogleフォーム→Claude Code→KJ法統合→コンセンサス評価のワークフロー。WSで体験した仕組みの「裏側」を解説
0:35–1:15ハンズオン: 実際にやってみる参加者同士で模擬授業。テーマを設定 → Googleフォームで意見収集 → Claude Codeで統合 → コンセンサス評価。教員自身が「AI統合ファシリテーター」を体験
1:15–1:35応用: 職員会議への導入を考える「この仕組みを職員会議に使うとしたら、最初のテーマは何か？」をグループで議論
1:35–1:50全体共有＋質疑職員会議での活用アイデアを3–4グループが共有
1:50–2:00チェックアウト＋次回までの課題授業でAI統合を1回試すか、職員会議の1テーマで試す

教員が持ち帰るもの

  • AI統合ファシリテーションのワークフロー（Googleフォーム→Claude Code→KJ法→コンセンサス）
  • 授業での活用イメージ（生徒の意見を統合して探究の方向性を決める等）
  • 職員会議への導入イメージ（最初の1テーマの候補）

ここで2つの土台が見える

土台内容
授業のAI活用第1回のAI壁打ち＋第2回のAI統合で、「個別最適」と「協働的な学び」の両方にAIを使える
職員会議のAI活用第2回のAI統合＋コンセンサス評価を職員会議に応用すれば、「50人中50人が参画する」意思決定が可能に

第3回: AI反転授業を設計する（オンライン2時間）

この回の目的

第1回（AI壁打ち）と第2回（AI統合）を組み合わせた「AI反転授業」を、自分の教科で実際に設計する。

事前準備

対象やること
参加教員授業またはHRでAI統合を1回試した体験を振り返る

当日のフロー

時間内容方法
0:00–0:15実践報告第2回後のAI統合の実践体験を3–4名が共有。うまくいったこと・つまずいたこと
0:15–0:35講義: AI反転授業の設計原理従来の反転授業（動画→教室で演習）をAI時代にアップデート。事前：AI壁打ちで考えを深掘り → 教室：AI統合で全員の考えを可視化 → 対話で深化 → 振り返り
0:35–1:20設計ワーク: 自分のAI反転授業を設計する1人1枚の設計シートに「単元名・事前課題（AI壁打ちの問い）・教室活動（AI統合のテーマ）・評価方法」を記入。Claude Codeに設計案を壁打ちさせてブラッシュアップ
1:20–1:40ペアレビュー＋改善隣の教員と設計案を交換しフィードバック。異教科のペアが望ましい（越境の視点）
1:40–1:55全体共有＋今後のスケジュール設計案を3–4名が共有。事例検討会のスケジュールを決定
1:55–2:00チェックアウト「来週からやること」を1人一言

教員が持ち帰るもの

  • 自分の教科のAI反転授業の設計案（1単元分）
  • 設計のレビューを受けた改善版
  • 事例検討会の日程と参加コミットメント

事例検討会（月1回・オンライン1時間・継続）

事例検討会の目的

研修で学んだことを「実践→共有→改善」のサイクルで定着させる。教員同士がAI活用の成功事例と失敗事例を持ち寄り、学び合うコミュニティを形成する。

フロー（月1回・1時間）

時間内容方法
0:00–0:05チェックイン1人一言
0:05–0:25事例発表（2名）各10分。「やったこと・生徒の反応・気づき・困っていること」。北田がリアルタイムでClaude Code構造化→GitHub公開
0:25–0:45対話発表を受けて全員で対話。「自分の教科ならこう応用できる」「こういう工夫はどうか」
0:45–0:55AI統合対話の内容をClaude Codeで統合。パターンを抽出し、「今月の学び」として可視化
0:55–1:00チェックアウト「来月試すこと」を1人一言

事例検討会が「文化」になる仕組み

仕組み内容
毎回GitHubに蓄積事例と対話の記録がURLで共有され、回を重ねるごとに「教員コミュニティの知恵」が蓄積される
パターン抽出複数回の事例からAIが共通パターンを抽出。「こういう使い方がうまくいく」という型が見えてくる
異教科の越境数学教員の事例が英語教員に転用される等、教科を越えた知恵の流通が自然に起きる
新しい教員の参入3回の研修を受けていない教員も、事例検討会から参加可能。先輩教員の実践を見て学ぶ
