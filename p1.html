P1: 教員向け研修プログラム

WS＋3回研修＋事例検討会で、「両利きの学校」の土台を作る

なぜ教員研修から始めるのか

「50人の先生がいても取り組むのは2-3人。30時間中30時間全部変わってほしい」
——馬場さん（長崎県教育委員会）

生徒の学びを変えるには、まず教員が変わる必要がある。しかし「AIを使いましょう」と言うだけでは変わらない。教員自身がAI共創を体験し、「これは使える」と実感することが出発点になる。

この研修で教員が体験するのは、生徒に育てたい7つのAI PROOFコンピテンシーそのものだ。教員が自ら体験していない力を、生徒に育てることはできない。

研修の回教員が体験するコンピテンシー
WS: 2040年問題の共有6. 合意形成と共創 — コンセンサス型で全員の声を統合する体験
第1回: AIアプリを作る1. 仕組み化と再投資 — AIで仕組みを作り、空いた時間で価値を創る体験
第2回: 意見をAIで統合する4. 知の形式知化と接続 — 多様な声を構造化し、知恵としてつなげる体験
第3回: AI反転授業を設計する5. 仮説駆動 — 仮説を立てて授業を設計し、実践で検証する体験
事例検討会: 実践の共有3. パターン発見 — 複数の事例から共通パターンを見出す体験
実証済みの方法

この研修プログラムは、かえつ有明高校（教員30名、3回＋事例研究会）と長崎県教育委員会（7名、導入版90分）で実施済み。長崎研修では「本当の意味での民主主義が実現していく夢を抱かせる話だった」（馬場さん）という評価を得ている。詳細はAbout Usを参照。

AI反転授業とは

事前にAI壁打ちで予習・深掘りし、教室では対話と深化に集中する授業形態。「教室で講義を聞く」時間を「教室で考え、議論する」時間に転換する。

P1が完成させる2つの土台——これが「両利きの学校」を実現するインフラになる:

  1. 授業のAI活用（深化の土台）: AI壁打ち（個別最適）＋AI統合（協働的な学び）で教科学習が変わる
  2. 職員会議のAI活用（組織変革の土台）: コンセンサス型意思決定で「50人中50人が参画する」学校経営が始まる

WS: 2040年問題の共有ワークショップ（2時間）

このWSの目的

「なぜAIを使うのか」の共通認識を教員全体で作る。2040年の社会像を自分ごととして考え、そこから逆算して「自分の授業をどう変えるか」の問いを持って帰る。

実施体制: プロセス設計（田原）、ダイアログファシリテーター、AIナレッジファシリテーターの3者で協働して実施。各ロールの担当者はAbout Usを参照。

事前準備

対象やることツール
参加教員プラネタリー・ラーニング構想の概要動画（15分程度）を視聴動画配信
参加教員AI壁打ちアプリで「2040年に高校教育はどう変わるべきか」について自分の考えを深掘りAI壁打ちアプリ
参加教員壁打ちで深掘りした考えをテキストで提出Googleフォーム
田原・北田提出されたテキストをClaude Codeで事前に構造化・分析Claude Code

当日のフロー

時間内容方法
0:00–0:10チェックイン1人30秒で「今の気持ち」を共有
0:10–0:30講義: 2040年の社会像ホワイトカラー中間層の縮小、新しい「はたらく人」の2類型（世界観ページ参照）
0:30–0:45グループ対話（3–4人）「2040年の社会で、自分が教えている教科は何のために存在するか？」
0:45–0:55意見入力スマホ/PCからGoogleフォームに入力:「自分の授業で最も変えるべきことは何か？」
0:55–1:15AI統合・可視化（ライブ）Claude Codeで全意見をリアルタイムKJ法統合。スクリーンに投影しながらクラスターが浮かび上がる過程を全員で見る
1:15–1:35コンセンサス評価統合結果から「学校変革の方向性」叩き台を生成 → 1/2/3で評価。AIエージェント（生徒・保護者・慎重派教員）の声も追加 → バージョンアップ
1:35–1:50全体共有バージョンアップされた方向性を全員で確認
1:50–2:00チェックアウト1人30秒で「持ち帰ること」を共有

このWSで教員が体験すること

体験意味
AI壁打ちで自分の考えを深掘りする事前課題で「AIと対話する」体験 = 反転授業の事前学習と同じ構造
全員の意見がAIで統合される「声の大きい人が通る」のではなく、全員の声が構造化される体験
コンセンサス評価でバージョンアップ批判が「提案を良くする力」になる体験 → 職員会議への導入イメージ
AIエージェントが不在の声を代弁する生徒・保護者・慎重派の視点が自動的に入る仕組み
WSの設計意図

このWS自体が、第1回〜第3回の研修で学ぶことの実体験版。教員は「教えられる」のではなく「体験する」ことで、研修への動機が生まれる。

第1回: AIアプリを作る（オンライン2時間）

この回の目的

教員が自分の授業で使える「AI壁打ちアプリ」を実際に作る。「AIを使える」から「AIアプリを作れる」への跳躍。
体験するコンピテンシー: 1. 仕組み化と再投資

実施体制: プロセス設計（田原）、ダイアログファシリテーター、AIナレッジファシリテーターの3者で協働して実施。

事前準備

WSで体験したAI壁打ちを、自分の授業テーマで1回やってみる。

当日のフロー

時間内容方法
0:00–0:10チェックイン＋事前課題の振り返りラウンド
0:10–0:30講義: AI壁打ちアプリの設計思想「答えを返すAI」（鏡）vs「問い返すAI」（回折格子）
0:30–1:10ハンズオン: 自分の授業用AI壁打ちアプリを作る自分の教科に最適化した壁打ちプロンプトを設計（例: 数学の証明、英語エッセイ、理科実験の仮説）
1:10–1:30ペアで試用作ったアプリを隣の教員に使ってもらいフィードバック
1:30–1:50改善＋共有フィードバックを反映して改善。3–4名が全体に共有
1:50–2:00チェックアウト＋次回までの課題次回までに自分の授業で1回使ってみる

教員が持ち帰るもの

  • 自分の教科用のAI壁打ちアプリ（プロンプト）
  • 生徒に使わせるための運用イメージ
  • 次回までに授業で1回使うという具体的なアクション

第2回: 多様な意見をAIで統合する（オンライン2時間）

この回の目的

授業やHRで、生徒の意見をAIで統合し、全員が参画する合意形成を実施できるようになる。これが職員会議への導入の土台にもなる。
体験するコンピテンシー: 4. 知の形式知化と接続 ＋ 6. 合意形成と共創

実施体制: プロセス設計（田原）、ダイアログファシリテーター、AIナレッジファシリテーターの3者で協働して実施。

事前準備

第1回で作ったAI壁打ちアプリを授業で使った経験を振り返り、メモにまとめる。

当日のフロー

時間内容方法
0:00–0:15実践報告第1回後に授業でAIを使った体験を3–4名が共有
0:15–0:35講義: AI統合ファシリテーションGoogleフォーム→Claude Code→KJ法統合→コンセンサス評価のワークフロー
0:35–1:15ハンズオン: 実際にやってみる参加者同士で模擬授業。テーマ設定→フォームで意見収集→Claude Codeで統合→コンセンサス評価
1:15–1:35応用: 職員会議への導入を考える「この仕組みを職員会議に使うとしたら、最初のテーマは何か？」をグループで議論
1:35–1:50全体共有＋質疑職員会議での活用アイデアを3–4グループが共有
1:50–2:00チェックアウト＋次回までの課題授業でAI統合を1回試すか、職員会議の1テーマで試す

教員が持ち帰るもの

  • AI統合ファシリテーションのワークフロー
  • 授業での活用イメージ＋職員会議への導入イメージ

ここで「両利き」の2つの土台が見える

授業のAI活用（深化の土台） 職員会議のAI活用（組織変革の土台）
第1回のAI壁打ち（個別最適）＋第2回のAI統合（協働的な学び）で、教科学習にAIを組み込める 第2回のAI統合＋コンセンサス評価を職員会議に応用すれば、全員参画の意思決定が可能に → P2へ

第3回: AI反転授業を設計する（オンライン2時間）

この回の目的

第1回（AI壁打ち）と第2回（AI統合）を組み合わせた「AI反転授業」を、自分の教科で実際に設計する。
体験するコンピテンシー: 5. 仮説駆動と事業開発

実施体制: プロセス設計（田原）、ダイアログファシリテーター、AIナレッジファシリテーターの3者で協働して実施。

事前準備

授業またはHRでAI統合を1回試した体験を振り返る。

当日のフロー

時間内容方法
0:00–0:15実践報告第2回後の実践体験を3–4名が共有
0:15–0:35講義: AI反転授業の設計原理事前: AI壁打ちで深掘り → 教室: AI統合で可視化 → 対話で深化 → 振り返り
0:35–1:20設計ワーク設計シートに「単元名・事前課題・教室活動・評価方法」を記入。Claude Codeに壁打ちさせてブラッシュアップ
1:20–1:40ペアレビュー＋改善隣の教員と設計案を交換しフィードバック。異教科のペアが望ましい（越境の視点）
1:40–1:55全体共有＋今後のスケジュール設計案を3–4名が共有。事例検討会のスケジュールを決定
1:55–2:00チェックアウト「来週からやること」を1人一言

教員が持ち帰るもの

  • 自分の教科のAI反転授業の設計案（1単元分）
  • ペアレビューを受けた改善版
  • 事例検討会の日程と参加コミットメント

事例検討会（月1回・オンライン1時間・継続）

事例検討会の目的

研修で学んだことを「実践→共有→改善」のサイクルで定着させる。教員同士がAI活用の成功事例と失敗事例を持ち寄り、学び合うコミュニティを形成する。
体験するコンピテンシー: 3. パターン発見 ＋ 2. 越境学習

実施体制: ダイアログファシリテーター＋AIナレッジファシリテーターが主導。プロセス設計（田原）は初回〜3回目まで参加、以降は必要に応じて。

フロー（月1回・1時間）

時間内容方法
0:00–0:05チェックイン1人一言
0:05–0:25事例発表（2名）各10分。「やったこと・生徒の反応・気づき・困っていること」。北田がリアルタイムでClaude Code構造化→GitHub公開
0:25–0:45対話「自分の教科ならこう応用できる」「こういう工夫はどうか」
0:45–0:55AI統合対話内容をClaude Codeで統合。「今月の学び」として可視化
0:55–1:00チェックアウト「来月試すこと」を1人一言

事例検討会が「文化」になる仕組み

仕組み内容
毎回GitHubに蓄積事例と対話の記録がURLで共有され、回を重ねるごとに「教員コミュニティの知恵」が蓄積される
パターン抽出複数回の事例からAIが共通パターンを抽出。「こういう使い方がうまくいく」という型が見えてくる
異教科の越境数学教員の事例が英語教員に転用される等、教科を越えた知恵の流通が自然に起きる
新しい教員の参入3回の研修を受けていない教員も、事例検討会から参加可能。先輩教員の実践を見て学ぶ

P1からStage 1へ — プラットフォーム展開の起点

第3回の研修で授業設計案を作った教員は、その設計案を自分の保管庫（Vault）に蓄積する。これがClaude Code＋ObsidanプラットフォームのStage 1（教員コアチーム）の始まりになる。

研修と環境導入が一体化している

P1は「研修プログラム」であると同時に「プラットフォーム導入の入口」。研修で「これは使える」と実感した教員が、そのままStage 1のコアチームになる。ツールを先に入れるのではなく、体験から始める。
